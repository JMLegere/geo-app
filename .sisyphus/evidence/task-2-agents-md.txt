AGENTS.MD SECTIONS
==================

1. Core Principles
   - Debug for root cause first
   - Prefer high-signal logs
   - Respect API constraints
   - Surface runtime values
   - Keep changes reversible

2. Architecture / Design
   - Runtime Shell (FogOfWorldApp)
   - Map Layer (MapLibre + Impeller)
   - Location Tracking (PlayerLocationTracker)
   - Geo Mapping (GeoReference, GeoUtils)
   - Fog State System (FogStateProvider, CellStateSystem)
   - Persistence Layer (LocalFirstPersistence, SyncQueue)
   - View Layer (CellTileView, FogOverlay, PlayerMarker)
   - Camera Controller (PlayerCameraFollower)

3. Directory Structure
   - lib/core/ (constants, models, services, utils)
   - lib/features/ (map, location, fog, persistence, species)
   - lib/shared/ (constants, theme, extensions)
   - test/ (core, features, fixtures)

4. Key Patterns
   - Riverpod Providers (StateNotifier, FutureProvider, StreamProvider, Family)
   - Local-First Persistence (SQLite, SyncQueue, conflict resolution)
   - Event-Driven State Transitions (OnCellStateChanged)
   - High-Signal Logging (GPS, tiles, fog state, persistence, sync)

5. Debugging Guide
   - Widget Tree Inspection (FogOverlay, CustomPaint, Impeller)
   - Riverpod Provider State Inspection (fogStateProvider, locationProvider)
   - Map Rendering Diagnostics (MapLibre, tiles, fog shader)
   - GPS Accuracy Logging (accuracy threshold, permission handling)
   - Persistence Diagnostics (SQLite, sync queue, conflicts)

6. Constraints
   - Scope Ceilings table (30 species, 5 biomes, 6 per biome, 5 fog levels, 3 rarity tiers, 2 seasons, etc.)
   - Forbidden Patterns (no monolithic bootstrap, no type-safety bypasses, no direct SQLite, no blocking I/O, no global state, no hardcoded constants, no platform-specific business logic)

7. Known Risks
   - Camera Sync (High) - map camera and player position drift
   - Platform View Composition (High) - MapLibre + Impeller flickering
   - GPS Accuracy Under Canopy (Medium) - accuracy degradation
   - iOS Permission Handling (Medium) - permission request and fallback
   - Offline-First Sync (Medium) - unbounded queue growth and conflicts

8. Debugging Checklist
   - Capture exact inputs/outputs
   - Apply 5-why analysis
   - Isolate the system
   - Check constraints
   - Verify reversibility
   - Log the fix

9. Future Work
   - Real tile provider (MBTiles/Mapbox API)
   - Species discovery mechanics
   - Seasonal variation
   - Multiplayer support
   - Analytics tracking
