SCENARIO: Step hydration skipped on web platform

Implementation:

  In rehydrateData():

      try {

          totalSteps: ref.read(playerProvider).totalSteps,

    }

  In hydrateAndStart() .whenComplete():

      ref.read(stepProvider.notifier).startLiveStream();

Web stub behavior (step_service_web.dart):
  - getCurrentStepCount() always returns 0
  - stepCountStream is Stream<int>.empty()
  - start() / stop() / dispose() are no-ops

Platform guard: kIsWeb from flutter/foundation.dart
  - On web: neither hydrate() nor startLiveStream() is called
  - On native: both are called in sequence (hydrate first, then startLiveStream)

Note: Even if web guard were removed, the web StepService stub would return
delta=0 (getCurrentStepCount=0, lastKnownStepCount=0 → delta=0). The guard
is belt-and-suspenders to avoid unnecessary async work on web.
