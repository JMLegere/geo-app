# Entity-Specific Data Flows

Complete lifecycle for each entity type: creation → persistence → sync → hydration.

---

## 1. Item Instance

### Creation (Discovery)

```
GameCoordinator.onItemDiscovered(DiscoveryEvent event, ItemInstance instance)
  ├─ instance.id = UUID v4 (generated by GameEngine)
  ├─ instance.definitionId = species ID (e.g., "fauna_vulpes_vulpes")
  ├─ instance.affixes = rolled affixes (from StatsService)
  ├─ instance.acquiredAt = DateTime.now()
  ├─ instance.acquiredInCellId = current cell ID
  ├─ instance.dailySeed = current daily seed (for server re-derivation)
  ├─ instance.status = 'active'
  └─ instance.badges = [] (or ['first_discovery'] if offline mode)
```

### Persistence (SQLite + Write Queue)

```
persistItemDiscovery(instance, userId)
  ├─ 1. SQLite write
  │   └─ itemRepo.addItem(instance, userId)
  │       └─ LocalItemInstanceTable.insert({
  │           id: instance.id,
  │           userId: userId,
  │           definitionId: instance.definitionId,
  │           displayName: instance.displayName,
  │           scientificName: instance.scientificName,
  │           categoryName: instance.category.name,
  │           rarityName: instance.rarity?.name,
  │           habitatsJson: instance.habitatsToJson(),
  │           continentsJson: instance.continentsToJson(),
  │           taxonomicClass: instance.taxonomicClass,
  │           affixes: instance.affixesToJson(),
  │           badgesJson: instance.badgesToJson(),
  │           parentAId: instance.parentAId,
  │           parentBId: instance.parentBId,
  │           acquiredAt: instance.acquiredAt,
  │           acquiredInCellId: instance.acquiredInCellId,
  │           dailySeed: instance.dailySeed,
  │           status: instance.status.name,
  │           // Denormalized enrichment (snapshotted at discovery)
  │           animalClassName: instance.animalClassName,
  │           foodPreferenceName: instance.foodPreferenceName,
  │           climateName: instance.climateName,
  │           brawn: instance.brawn,
  │           wit: instance.wit,
  │           speed: instance.speed,
  │           sizeName: instance.sizeName,
  │           iconUrl: instance.iconUrl,
  │           artUrl: instance.artUrl,
  │           // Cell properties denorm
  │           cellHabitatName: instance.cellHabitatName,
  │           cellClimateName: instance.cellClimateName,
  │           cellContinentName: instance.cellContinentName,
  │           // Location hierarchy denorm
  │           locationDistrict: instance.locationDistrict,
  │           locationCity: instance.locationCity,
  │           locationState: instance.locationState,
  │           locationCountry: instance.locationCountry,
  │           locationCountryCode: instance.locationCountryCode,
  │         })
  │
  └─ 2. Write queue enqueue (only if SQLite succeeds)
      └─ queueProcessor.enqueue(
           entityType: WriteQueueEntityType.itemInstance,
           entityId: instance.id,
           operation: WriteQueueOperation.upsert,
           payload: jsonEncode({
             'id': instance.id,
             'definition_id': instance.definitionId,
             'display_name': instance.displayName,
             'scientific_name': instance.scientificName,
             'category_name': instance.category.name,
             'rarity_name': instance.rarity?.name,
             'habitats_json': instance.habitatsToJson(),
             'continents_json': instance.continentsToJson(),
             'taxonomic_class': instance.taxonomicClass,
             'affixes': instance.affixesToJson(),
             'badges_json': instance.badgesToJson(),
             'parent_a_id': instance.parentAId,
             'parent_b_id': instance.parentBId,
             'acquired_at': instance.acquiredAt.toIso8601String(),
             'acquired_in_cell_id': instance.acquiredInCellId,
             'daily_seed': instance.dailySeed,
             'status': instance.status.name,
           }),
           userId: userId,
         )
         └─ LocalWriteQueueTable.insert(status='pending', attempts=0)
         └─ Auto-schedule flush (5s debounce)
```

### Sync (Write Queue → Supabase)

```
QueueProcessor.flush()
  └─ For each pending itemInstance entry:
      ├─ SupabasePersistence.upsertItemInstance({
      │   id: entry.entityId,
      │   userId: userId,
      │   definitionId: payload['definition_id'],
      │   affixes: payload['affixes'],
      │   // ... all fields from payload
      │ })
      │   └─ Supabase `item_instances` table.upsert()
      │       └─ Server validates:
      │           ├─ User owns this item (RLS)
      │           ├─ Definition exists
      │           ├─ Encounter is valid (re-derive from seed + cellId)
      │           └─ Award first-discovery badge if applicable
      │
      ├─ On success:
      │   └─ queueRepo.deleteEntry(id)
      │       └─ LocalWriteQueueTable.delete(id)
      │
      ├─ On retryable error (network, Supabase down):
      │   └─ queueRepo.incrementAttempts(id, error)
      │       └─ LocalWriteQueueTable.update(attempts++, lastError=error)
      │       └─ If attempts >= 5:
      │           └─ queueRepo.markRejected(id, error)
      │               └─ LocalWriteQueueTable.update(status='rejected')
      │
      └─ On validation rejection (server says invalid):
          └─ queueRepo.markRejected(id, error)
              └─ LocalWriteQueueTable.update(status='rejected')
              └─ SyncNotifier.processRejections()
                  ├─ itemsProvider.removeItem(entry.entityId)
                  ├─ itemRepo.deleteItem(entry.entityId)
                  │   └─ LocalItemInstanceTable.delete(id)
                  └─ queueRepo.deleteEntry(id)
                      └─ LocalWriteQueueTable.delete(id)
```

### Hydration (Supabase → SQLite)

```
hydrateFromSupabase(userId)
  └─ persistence.fetchItemInstances(userId)
      └─ Supabase `item_instances` table.select().eq('user_id', userId)
          └─ For each row:
              ├─ Parse into ItemInstance domain object
              └─ itemRepo.upsertItem(instance, userId)
                  └─ LocalItemInstanceTable.insertOnConflictUpdate({
                      id: row['id'],
                      userId: userId,
                      definitionId: row['definition_id'],
                      // ... all fields from row
                    })
                    └─ If item already exists locally: update with server values
                    └─ If new: insert
```

### Display (SQLite → UI)

```
itemsProvider (NotifierProvider<ItemsNotifier, ItemsState>)
  ├─ On startup: itemsNotifier.loadItems(items)
  │   └─ itemRepo.getItemsByUser(userId)
  │       └─ LocalItemInstanceTable.select().eq('user_id', userId)
  │           └─ Convert LocalItemInstance → ItemInstance domain objects
  │
  ├─ On discovery: itemsNotifier.addItem(instance)
  │   └─ state = {...state, items: [...state.items, instance]}
  │
  ├─ On rejection: itemsNotifier.removeItem(itemId)
  │   └─ state = {...state, items: state.items.where((i) => i.id != itemId)}
  │
  └─ Consumed by:
      ├─ PackScreen (inventory view)
      ├─ SanctuaryScreen (collection view)
      └─ DiscoveryScreen (toast notification)
```

## 2. Cell Progress

### Creation (Cell Visit)

```
GameCoordinator.onCellVisited(String cellId)
  └─ persistCellVisit(cellId, userId)
      ├─ 1. Check if first visit
      │   └─ cellProgressRepo.read(userId, cellId)
      │       └─ LocalCellProgressTable.select()
      │           .where(userId=userId AND cellId=cellId)
      │
      ├─ 2a. First visit: create new record
      │   └─ cellProgressRepo.create(
      │       id: uuid.v4(),
      │       userId: userId,
      │       cellId: cellId,
      │       fogState: FogState.present,
      │       visitCount: 1,
      │       lastVisited: DateTime.now(),
      │     )
      │       └─ LocalCellProgressTable.insert({
      │           id: progressId,
      │           userId: userId,
      │           cellId: cellId,
      │           fogState: 'present',
      │           visitCount: 1,
      │           lastVisited: now,
      │           createdAt: now,
      │           updatedAt: now,
      │         })
      │
      └─ 2b. Returning visit: increment visit count
          └─ cellProgressRepo.incrementVisitCount(userId, cellId)
              └─ LocalCellProgressTable.update(
                  visitCount = visitCount + 1,
                  lastVisited = DateTime.now(),
                )
```

### Persistence (SQLite + Write Queue)

```
persistCellVisit(cellId, userId)
  ├─ 1. SQLite write (see above)
  │
  └─ 2. Write queue enqueue (only if SQLite succeeds)
      └─ queueProcessor.enqueue(
           entityType: WriteQueueEntityType.cellProgress,
           entityId: '$userId:$cellId',
           operation: WriteQueueOperation.upsert,
           payload: jsonEncode({
             'cell_id': cellId,
             'fog_state': 'present',
             'visit_count': visitCount,  // Current DB value
             'distance_walked': distanceWalked,  // Current DB value
             'last_visited': now.toIso8601String(),
           }),
           userId: userId,
         )
         └─ LocalWriteQueueTable.insert(status='pending', attempts=0)
         └─ Auto-schedule flush (5s debounce)
```

**Key:** Payload uses current DB values (visitCount, distanceWalked) to avoid stale defaults if SQLite write succeeded but enqueue is delayed.

### Sync (Write Queue → Supabase)

```
QueueProcessor.flush()
  └─ For each pending cellProgress entry:
      ├─ SupabasePersistence.upsertCellProgress({
      │   userId: userId,
      │   cellId: payload['cell_id'],
      │   fogState: payload['fog_state'],
      │   visitCount: payload['visit_count'],
      │   distanceWalked: payload['distance_walked'],
      │   lastVisited: DateTime.parse(payload['last_visited']),
      │ })
      │   └─ Supabase `cell_progress` table.upsert(
      │       onConflict: 'user_id,cell_id'
      │     )
      │
      ├─ On success:
      │   └─ queueRepo.deleteEntry(id)
      │
      ├─ On retryable error:
      │   └─ queueRepo.incrementAttempts(id, error)
      │       └─ If attempts >= 5: queueRepo.markRejected(id, error)
      │
      └─ On validation rejection:
          └─ queueRepo.markRejected(id, error)
              └─ SyncNotifier.processRejections()
                  └─ Log only (server is authoritative, next sync reconciles)
```

### Hydration (Supabase → SQLite)

```
hydrateFromSupabase(userId)
  └─ persistence.fetchCellProgress(userId)
      └─ Supabase `cell_progress` table.select().eq('user_id', userId)
          └─ For each row:
              └─ cellProgressRepo.create(
                  id: row['id'] ?? '${userId}_${cellId}',
                  userId: userId,
                  cellId: row['cell_id'],
                  fogState: FogState.fromString(row['fog_state']),
                  visitCount: row['visit_count'],
                  distanceWalked: row['distance_walked'],
                  lastVisited: DateTime.parse(row['last_visited']),
                )
                  └─ LocalCellProgressTable.insert(...)
                  └─ Yield every 50 rows to prevent UI jank
```

### Display (SQLite → UI)

```
fogResolver (FogStateResolver)
  ├─ On startup: fogResolver.loadVisitedCells(visitedCellIds)
  │   └─ cellProgressRepo.readByUser(userId)
  │       └─ LocalCellProgressTable.select().eq('user_id', userId)
  │           └─ Extract cellIds where fogState in ['present', 'explored']
  │
  ├─ On cell visit: fogResolver.onLocationUpdate(position, visitedCellIds)
  │   └─ Compute fog state for all cells based on:
  │       ├─ Player position
  │       ├─ Distance to cell center
  │       └─ Whether cell is in visitedCellIds
  │
  └─ Consumed by:
      ├─ MapScreen (fog overlay rendering)
      └─ PackScreen (collection filtering)
```

## 3. Player Profile

### Creation (First Login)

```
gameCoordinatorProvider.hydrateAndStart(userId)
  └─ rehydrateData(userId)
      └─ profileRepo.read(userId)
          └─ LocalPlayerProfileTable.select().eq('id', userId)
              └─ If null: profile doesn't exist yet
                  └─ playerProvider.loadProfile(
                      cellsObserved: 0,
                      totalDistanceKm: 0.0,
                      currentStreak: 0,
                      longestStreak: 0,
                    )
                      └─ state = PlayerState(...)
```

### Persistence (SQLite + Write Queue)

```
PlayerNotifier state change (cells, distance, streaks)
  └─ gameCoordinatorProvider (ref.listen on playerProvider)
      ├─ Debounce 5s (accumulate rapid changes)
      └─ persistProfileState(userId, playerState)
          ├─ 1. SQLite write
          │   └─ profileRepo.read(userId)
          │       └─ If exists: profileRepo.update({
          │           currentStreak: playerState.currentStreak,
          │           longestStreak: playerState.longestStreak,
          │           totalDistanceKm: playerState.totalDistanceKm,
          │           currentSeason: Season.fromDate(DateTime.now()).name,
          │           hasCompletedOnboarding: playerState.hasCompletedOnboarding,
          │           lastLat: currentPos?.lat,
          │           lastLon: currentPos?.lon,
          │           totalSteps: playerState.totalSteps,
          │           lastKnownStepCount: playerState.lastKnownStepCount,
          │         })
          │           └─ LocalPlayerProfileTable.update(...)
          │
          │       └─ If new: profileRepo.create({
          │           userId: userId,
          │           displayName: 'Explorer',
          │           currentStreak: playerState.currentStreak,
          │           // ... other fields
          │         })
          │           └─ LocalPlayerProfileTable.insert(...)
          │
          └─ 2. Write queue enqueue (only if SQLite succeeds)
              └─ queueProcessor.enqueue(
                   entityType: WriteQueueEntityType.profile,
                   entityId: userId,
                   operation: WriteQueueOperation.upsert,
                   payload: jsonEncode({
                     'display_name': 'Explorer',
                     'current_streak': playerState.currentStreak,
                     'longest_streak': playerState.longestStreak,
                     'total_distance_km': playerState.totalDistanceKm,
                     'current_season': season.name,
                     'has_completed_onboarding': playerState.hasCompletedOnboarding,
                     'total_steps': playerState.totalSteps,
                     'last_known_step_count': playerState.lastKnownStepCount,
                     'last_lat': lastLat,
                     'last_lon': lastLon,
                   }),
                   userId: userId,
                 )
                 └─ LocalWriteQueueTable.insert(status='pending', attempts=0)
                 └─ Auto-schedule flush (5s debounce)
```

### Sync (Write Queue → Supabase)

```
QueueProcessor.flush()
  └─ For each pending profile entry:
      ├─ SupabasePersistence.upsertProfile({
      │   userId: userId,
      │   displayName: payload['display_name'],
      │   currentStreak: payload['current_streak'],
      │   longestStreak: payload['longest_streak'],
      │   totalDistanceKm: payload['total_distance_km'],
      │   currentSeason: payload['current_season'],
      │   hasCompletedOnboarding: payload['has_completed_onboarding'],
      │ })
      │   └─ Supabase `profiles` table.upsert()
      │
      ├─ On success:
      │   └─ queueRepo.deleteEntry(id)
      │
      ├─ On retryable error:
      │   └─ queueRepo.incrementAttempts(id, error)
      │       └─ If attempts >= 5: queueRepo.markRejected(id, error)
      │
      └─ On validation rejection:
          └─ queueRepo.markRejected(id, error)
              └─ SyncNotifier.processRejections()
                  └─ Log only (server is authoritative, next sync reconciles)
```

### Hydration (Supabase → SQLite)

```
hydrateFromSupabase(userId)
  └─ persistence.fetchProfile(userId)
      └─ Supabase `profiles` table.select().eq('id', userId).maybeSingle()
          └─ If found:
              └─ profileRepo.create({
                  userId: userId,
                  displayName: row['display_name'] ?? 'Explorer',
                  currentStreak: row['current_streak'] ?? 0,
                  longestStreak: row['longest_streak'] ?? 0,
                  totalDistanceKm: row['total_distance_km'] ?? 0.0,
                  currentSeason: row['current_season'] ?? 'summer',
                  hasCompletedOnboarding: row['has_completed_onboarding'] ?? false,
                })
                  └─ LocalPlayerProfileTable.insert(...)
```

### Display (SQLite → UI)

```
playerProvider (NotifierProvider<PlayerNotifier, PlayerState>)
  ├─ On startup: playerNotifier.loadProfile(...)
  │   └─ profileRepo.read(userId)
  │       └─ LocalPlayerProfileTable.select().eq('id', userId)
  │           └─ Convert LocalPlayerProfile → PlayerState
  │
  ├─ On cell visit: playerNotifier.incrementCellsObserved()
  │   └─ state = state.copyWith(cellsObserved: state.cellsObserved + 1)
  │
  ├─ On distance update: playerNotifier.addDistance(km)
  │   └─ state = state.copyWith(totalDistanceKm: state.totalDistanceKm + km)
  │
  └─ Consumed by:
      ├─ HomeScreen (stats display)
      ├─ MapScreen (distance counter)
      └─ OnboardingScreen (completion flag)
```

## 4. Cell Properties

### Creation (Cell Becomes Adjacent)

```
GameCoordinator.onCellPropertiesResolved(CellProperties properties)
  └─ persistCellProperties(properties, userId)
      ├─ 1. SQLite write
      │   └─ cellPropertyRepo.upsert(properties)
      │       └─ LocalCellPropertiesTable.upsert({
      │           cellId: properties.cellId,
      │           habitats: jsonEncode(properties.habitats.map((h) => h.name)),
      │           climate: properties.climate.name,
      │           continent: properties.continent.name,
      │           locationId: properties.locationId,
      │           createdAt: DateTime.now(),
      │         })
      │
      └─ 2. Write queue enqueue (only if SQLite succeeds)
          └─ queueProcessor.enqueue(
               entityType: WriteQueueEntityType.cellProperties,
               entityId: properties.cellId,
               operation: WriteQueueOperation.upsert,
               payload: jsonEncode({
                 'cell_id': properties.cellId,
                 'habitats': properties.habitats.map((h) => h.name).toList(),
                 'climate': properties.climate.name,
                 'continent': properties.continent.name,
                 'location_id': properties.locationId,
               }),
               userId: userId,
             )
             └─ LocalWriteQueueTable.insert(status='pending', attempts=0)
             └─ Auto-schedule flush (5s debounce)
```

**Key:** Cell properties are globally shared (no userId in SQLite), but write queue still needs userId for routing.

### Sync (Write Queue → Supabase)

```
QueueProcessor.flush()
  └─ For each pending cellProperties entry:
      ├─ SupabasePersistence.upsertCellProperties({
      │   cellId: payload['cell_id'],
      │   habitats: payload['habitats'],
      │   climate: payload['climate'],
      │   continent: payload['continent'],
      │   locationId: payload['location_id'],
      │ })
      │   └─ Supabase `cell_properties` table.upsert()
      │
      ├─ On success:
      │   └─ queueRepo.deleteEntry(id)
      │
      ├─ On retryable error:
      │   └─ queueRepo.incrementAttempts(id, error)
      │       └─ If attempts >= 5: queueRepo.markRejected(id, error)
      │
      └─ On validation rejection:
          └─ queueRepo.markRejected(id, error)
              └─ SyncNotifier.processRejections()
                  └─ Log only (server is authoritative, next sync reconciles)
```

### Hydration (Supabase → SQLite)

```
hydrateFromSupabase(userId)
  └─ persistence.fetchCellProperties(visitedCellIds)
      └─ Supabase `cell_properties` table.select().in('cell_id', visitedCellIds)
          └─ For each row:
              └─ cellPropertyRepo.upsert(CellProperties(
                  cellId: row['cell_id'],
                  habitats: (row['habitats'] as List)
                      .map((h) => Habitat.fromString(h.toString()))
                      .toSet(),
                  climate: Climate.fromString(row['climate']),
                  continent: Continent.fromString(row['continent']),
                  locationId: row['location_id'],
                  createdAt: DateTime.now(),
                ))
                  └─ LocalCellPropertiesTable.upsert(...)
                  └─ Yield every 50 rows to prevent UI jank
```

### Display (SQLite → UI)

```
coordinator.cellPropertiesCache (Map<String, CellProperties>)
  ├─ On startup: coordinator.loadCellProperties(propsMap)
  │   └─ cellPropertyRepo.getAll()
  │       └─ LocalCellPropertiesTable.select()
  │           └─ Convert LocalCellProperties → CellProperties
  │
  ├─ On cell resolution: coordinator.loadCellProperties({cellId: props})
  │   └─ In-memory cache update (no SQLite write yet)
  │
  └─ Consumed by:
      ├─ DiscoveryService (species loot table selection)
      ├─ FogOverlayController (detection zone rendering)
      └─ LocationEnrichmentService (district lookup)
```

## Key Principles

1. **SQLite first, then queue.** All writes go to SQLite first. Only after SQLite succeeds do we enqueue for Supabase.

2. **Payload uses current DB values.** For cell progress and profile, the write queue payload uses current DB values (not the in-memory state) to avoid stale defaults.

3. **Rejection rollback is item-only.** Only items are rolled back locally on rejection. Cell progress and profile are not rolled back (server is authoritative).

4. **Hydration doesn't re-hydrate providers.** Phase 2 (Supabase) runs in background without updating providers (avoids race with in-flight discoveries).

5. **Cell properties are globally shared.** No userId in SQLite table, but write queue still needs userId for routing.

6. **Denormalization is one-way.** Item instances snapshot species data at discovery time. Server-side enrichment changes don't propagate to existing items.

7. **Batching prevents UI jank.** Hydration loops yield every 50 rows. Cell property persistence batches 5 writes per batch.

8. **Debouncing prevents hammering SQLite.** Profile writes are debounced to 5s to accumulate rapid changes and avoid hammering IndexedDB-backed SQLite on iOS.
